## Quick Start

BaGency requires no installation, no build process, and no server-side setup.

### Option 1: Open Directly

```bash
# Simply open any HTML file in your browser
open index.html        # macOS
xdg-open index.html    # Linux
start index.html       # Windows
```

### Option 2: Local Server (recommended for development)

```bash
# Python
python3 -m http.server 8000

# Node.js (npx)
npx serve .

# PHP
php -S localhost:8000
```

Then navigate to `http://localhost:8000` in your browser.

### Option 3: VS Code Live Server

Install the **Live Server** extension in VS Code, right-click on `index.html`, and select **"Open with Live Server"**.

### Hosting

BaGency works on **any static hosting platform**:

- **Netlify** — Drag and drop the folder
- **Vercel** — `vercel --prod`
- **GitHub Pages** — Push to a `gh-pages` branch
- **AWS S3** — Upload as a static site
- **Shared hosting** — Upload via FTP/cPanel File Manager

> **Note**: CDN dependencies (Tailwind CSS, Google Fonts, Lucide Icons) require an internet connection. For offline use, see the self-contained `landing.html` as a reference for inlining these assets.

### Typography

| Font | Family | Weights | Usage |
|------|--------|---------|-------|
| **Inter** | `sans-serif` | 300, 400, 500, 600, 700, 800, 900 | Body text, navigation, UI labels, form inputs |
| **Space Grotesk** | `display` | 400, 500, 600, 700 | Headlines, hero text, stats, pricing figures, blog headings |

**Type Scale**: `text-xs` (12px) → `text-7xl` (72px) via Tailwind utility classes

**Line Heights**: Body text uses `leading-relaxed` (1.625), long-form content uses `leading-loose` (1.8)

### Glassmorphism

BaGency's signature visual effect uses `backdrop-filter: blur()` with semi-transparent backgrounds:

```css
/* Light mode */
.glass-card {
    background: rgba(255, 255, 255, 0.7);
    backdrop-filter: blur(20px);
    border: 1px solid rgba(255, 255, 255, 0.3);
}

/* Dark mode */
.dark .glass-card {
    background: rgba(30, 30, 40, 0.7);
    border: 1px solid rgba(255, 255, 255, 0.1);
}
```

### Dark Mode

The dark mode system uses Tailwind's `class` strategy:

1. **Detection order**: `localStorage` preference → `prefers-color-scheme: dark` → light (default)
2. **Activation**: Adds `.dark` class to `<html>` element
3. **Persistence**: Saves to `localStorage` key on manual toggle
4. **Transitions**: Smooth 0.3s easing on background, color, border, and shadow properties

## Customization

### Changing Colors

All brand colors are defined in **one file**: `js/tailwind-config.js`

```javascript
// In tailwind.config.theme.extend.colors:
primary: {
    500: '#22c55e',  // ← Change this to your brand color
    600: '#16a34a',
    // ... adjust the full scale to match
},
accent: {
    500: '#8b5cf6',  // ← Change this to your secondary color
}
```

> **Tip**: Use a tool like [UIColors.app](https://uicolors.app) or [Tailwind Shade Generator](https://www.tailwindshades.com/) to generate a complete shade scale from a single hex value.

Also update the hardcoded values in `css/style.css`:
- Button gradients (`.btn-primary`, `.btn-secondary`)
- Box shadows (`.btn-primary:hover`, `.glow`, `.glow-strong`)
- Mesh gradient orbs (`.mesh-gradient::before/::after`)
- Status pulse (`.status-pulse::before`)
- Back to top button (`#back-to-top`)
- Scrollbar thumb (`::-webkit-scrollbar-thumb`)
- Avatar gradient border (`.avatar-gradient`)
- Popular pricing card border and shadow

### Swapping Fonts

1. Replace the Google Fonts `<link>` tags in each HTML file's `<head>`:

```html
<!-- Old -->
<link href="https://fonts.googleapis.com/css2?family=Inter:wght@300;400;500;600;700;800;900&family=Space+Grotesk:wght@400;500;600;700&display=swap" rel="stylesheet">

<!-- New — replace with your preferred fonts -->
<link href="https://fonts.googleapis.com/css2?family=YourFont:wght@400;500;600;700&display=swap" rel="stylesheet">
```

2. Update the font families in `js/tailwind-config.js`:

```javascript
fontFamily: {
    sans: ['YourFont', 'sans-serif'],
    display: ['YourDisplayFont', 'sans-serif'],
}
```

3. Update `css/style.css` base styles:

```css
body { font-family: 'YourFont', sans-serif; }
```

### Adding Icons

BaGency uses **Lucide Icons** via CDN. Browse the full icon set at [lucide.dev/icons](https://lucide.dev/icons).

```html
<!-- Add any icon -->
<i data-lucide="icon-name" class="w-5 h-5"></i>

<!-- With color -->
<i data-lucide="rocket" class="w-5 h-5 text-primary-500"></i>

<!-- With size -->
<i data-lucide="arrow-right" class="w-4 h-4 mr-2"></i>
```

Icons are automatically initialized by `lucide.createIcons()` at the end of `main.js`.

> **Note**: Lucide does not include brand/social media icons (Twitter, LinkedIn, etc.). Social icons in the footer use inline SVGs.

## Animations

### Scroll-Based Reveals

Triggered by `IntersectionObserver` in `main.js`. Elements start invisible and transition to their final position when scrolled into view.

| Class | Effect | CSS |
|-------|--------|-----|
| `.reveal` | Fade up from 30px below | `translateY(30px) → translateY(0)` |
| `.reveal-left` | Slide in from 50px left | `translateX(-50px) → translateX(0)` |
| `.reveal-right` | Slide in from 50px right | `translateX(50px) → translateX(0)` |
| `.reveal-scale` | Scale up from 90% with fade | `scale(0.9) → scale(1)` |

All use `cubic-bezier(0.175, 0.885, 0.32, 1.275)` — a spring overshoot curve.

Stagger delays via inline `style="transition-delay: 0.1s"` on child elements.

### CSS Keyframes

| Name | Duration | Effect | Usage |
|------|----------|--------|-------|
| `float` | 6s | ±20px vertical bob | Decorative shapes, hero elements |
| `glowPulse` | 2s | Shadow intensity + scale pulse | CTA button glow halos |
| `typing` | 1.4s | Opacity blink | Chatbot typing dots |
| `slideUp` | 0.3s | 30px below → origin | Chat messages |
| `fadeIn` | 0.5s | Opacity 0→1 | Generic entrance |
| `bounceSubtle` | 3s | ±5px vertical bounce | Floating trust badges |
| `scaleIn` | 0.5s | Scale 0.9→1 with fade | Metric badges |
| `slideInLeft` | 0.6s | 50px left → origin | Timeline items |
| `slideInRight` | 0.6s | 50px right → origin | Timeline items |
| `shimmer` | 3s | Background-position sweep | Trust bar, loading states |
| `pulse` | 2s | Scale 1→1.5 with fade | Status indicators |

## Interactive Components

### ROI Calculator
**Location**: `pricing.html`  
**How it works**: Four range sliders (monthly leads, hours saved, hourly rate, software costs) feed into a real-time calculation. Annual savings, hours saved, and ROI percentage update instantly.

### AI Chatbot Demo
**Location**: `index.html`  
**How it works**: Simulated chatbot with predefined script flow, typing indicators (3 bouncing dots), quick-reply buttons, and keyword-matched responses. Responses defined in `data.js` → `chatbotScript` and `chatbotResponses`.

### Pricing Toggle
**Location**: `pricing.html`  
**How it works**: Monthly/yearly toggle updates all three pricing cards. Uses `cloneNode(true)` to swap DOM content, then re-initializes Lucide icons and event listeners.

### Testimonial Carousel
**Location**: `index.html`  
**How it works**: Horizontal flex container translated via `transform: translateX()`. Prev/next buttons and dot indicators control the current slide index. Each card shows rating, quote, metric badge, and author info.

### FAQ Accordion
**Location**: `index.html`, `pricing.html`, `faq.html`  
**How it works**: Click-to-toggle with smooth `max-height` CSS transition (0→500px). Chevron icon rotates 180°. FAQ page adds categorized tab filtering.

### Contact & Booking Forms
**Location**: `contact.html`  
**How it works**: Form submissions are validated and logged to console. On success, the submit button transforms to a checkmark confirmation with green background, auto-resets after 3 seconds.

## Architecture

### Data Layer (`js/data.js`)

All dynamic content is stored as plain JavaScript objects/arrays. No JS frameworks, no JSON parsing — just editable JS:

```
data.js
├── servicesData[6]       → Service cards (icon, title, description, features)
├── processData[4]        → Process timeline (step, title, description)
├── pricingData[3]        → Pricing plans (name, price, features, popular flag)
├── caseStudiesData[4]    → Case studies (client, metrics, description)
├── testimonialsData[6]   → Testimonials (name, quote, rating, company)
├── faqData[8]            → FAQ items (question, answer, category)
├── faqCategories[4]      → FAQ tab groups (name, items)
├── chatbotScript[2]      → Chatbot conversation flow
├── chatbotResponses[4]   → Chatbot keyword → response mapping
├── teamData[6]           → Team members (name, role, bio, social)
└── blogData[6]           → Blog posts (title, excerpt, category, author)
```

### Render Layer (`js/main.js`)

Render functions check for container elements by ID. If the element doesn't exist on the current page, the function exits gracefully:

```
main.js
├── initTheme()             → Dark mode detection + toggle
├── initNavbar()            → Scroll-based frosted glass effect
├── initMobileMenu()        → Hamburger toggle + Home accordion
├── initBackToTop()         → Show/hide button on scroll
├── renderServices()        → #services-grid → index, services
├── renderProcess()         → #process-timeline → index, services
├── renderPricingCards()    → #pricing-cards → pricing
├── renderCaseStudies()     → #case-studies-mini → index
├── renderCaseStudiesFull() → #case-studies-full → case-studies
├── renderTestimonials()    → #testimonials-track → index
├── renderFAQ()             → #faq-container → index
├── renderFAQCategories()   → #faq-categories → faq
├── renderTeam()            → #team-grid → about
├── renderBlogGrid()        → #blog-grid → blog
├── initTestimonialSlider() → Carousel prev/next/dots
├── initFAQ()               → Accordion toggle handlers
├── initCalculator()        → ROI slider → live calculation
├── initChatbot()           → Chat message flow
├── initBookingForm()       → Booking form submission
├── initContactForm()       → Contact form submission
├── initScrollReveal()      → IntersectionObserver → .revealed
└── initCounters()          → Animated number counting
```

### Initialization Flow

```
HTML loads
  ↓
<head> loads: Tailwind CDN → tailwind-config.js → Google Fonts → Lucide CDN → style.css
  ↓
<body> renders
  ↓
<script data.js>   ← Data layer loaded first
  ↓
<script main.js>   ← DOMContentLoaded fires:
  ↓
  1. initTheme()           ← Prevents flash of wrong theme
  2. initNavbar()          ← Scroll listener for glass effect
  3. initMobileMenu()      ← Hamburger + Home accordion
  4. initBackToTop()       ← Scroll show/hide
  5. Render functions      ← Inject dynamic content into containers
  6. Component inits       ← Slider, FAQ, calculator, chatbot, forms
  7. initScrollReveal()    ← IntersectionObserver (after DOM is complete)
  8. initCounters()        ← Animated number counters
  9. lucide.createIcons()  ← Last — scans all dynamic HTML for icons
```

## Responsive Design

Built mobile-first with progressive enhancement across 5 breakpoints:

| Breakpoint | Width | Layout |
|------------|-------|--------|
| **Mobile** | 320–639px | Single column, hamburger menu, stacked cards, reduced animations |
| **Small tablet** | 640–767px | Two-column grids begin, enhanced spacing |
| **Tablet** | 768–1023px | Two-column layouts, visible decorative elements |
| **Desktop** | 1024–1279px | Full multi-column, expanded navigation with dropdowns |
| **Large desktop** | 1280–1535px | Max-width container with generous padding |
| **Ultrawide** | 1536px+ | Full-width utilization with max-width constraints |

**Touch optimization**: All interactive elements have minimum 44px tap targets. Dropdowns, accordions, sliders, and carousels work with both mouse and touch input.

## Performance

| Optimization | Technique |
|-------------|-----------|
| **Font loading** | `preconnect` hints + `display=swap` to prevent FOIT |
| **Icon loading** | Lucide loaded via CDN with deferred initialization |
| **Scroll handlers** | Passive event listeners for scroll-based effects |
| **Canvas** | Particle count reduced on mobile viewports |
| **Animations** | `transform` and `opacity` only for GPU compositing |
| **CSS** | No unused styles — each class serves a purpose |
| **JS** | Graceful null checks — functions skip if containers don't exist |
