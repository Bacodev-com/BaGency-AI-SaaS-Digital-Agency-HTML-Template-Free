BaGency Documentation

Everything you need to set up, customize, and deploy your AI Automation Agency template. Built with Tailwind CSS, Lucide Icons, and vanilla JavaScript.

Introduction

BaGency is a premium, multi-page HTML template designed for AI automation agencies, SaaS startups, and digital service businesses. It features a modern glassmorphism design system, three homepage variants, fully working interactive components, and class-based dark mode — all without any build tools or frameworks.

Key Features

  • 3 Homepage Variants — Classic, Premium (Bento Grid), and Animated styles
  • 16 Pre-built Pages — About, Services, Case Studies, Pricing, Blog, FAQ, Contact, and legal pages
  • Glassmorphism Design — Modern frosted-glass cards and overlays
  • Dark Mode — Class-based toggle with localStorage persistence
  • Fully Responsive — Mobile-first, works on all screen sizes
  • Interactive Components — ROI calculator, chatbot demo, testimonial slider, FAQ accordion, billing toggle
  • Centralized Data — All content lives in js/data.js for easy editing
  • Scroll Reveal Animations — IntersectionObserver-powered entrance effects
  • No Build Tools — Pure HTML, CSS, and JavaScript — just open in a browser

Quick Start

Get BaGency up and running in three simple steps. No Node.js, npm, or build tools required.

1

Download & Extract

Download the template ZIP file and extract it to a folder on your computer. You'll see the bagency/ directory containing all files.

2

Upload or Open Locally

You have two options:

  • Local preview: Open index.html directly in your browser, or use a local server like VS Code Live Server.
  • Upload to server: Upload the entire bagency/ folder to your web server via FTP, SFTP, or your hosting panel's file manager.
3

Open in Browser

Navigate to https://yourdomain.com/bagency/ (or your local path) and you'll see the homepage. That's it — you're ready to customize!

For the best development experience, use VS Code with the Live Server extension. Right-click index.html and select "Open with Live Server" to get hot-reloading on file changes.

How It Works

BaGency uses a data-driven architecture. Content such as services, pricing plans, testimonials, and FAQ items are defined as JavaScript objects in js/data.js. The js/main.js file contains render functions that inject this data into the HTML on page load. This means you can change most content by editing data.js without touching HTML structure.

Static pages (About, legal pages, blog single) contain their content directly in the HTML for more flexible layouts.

Fonts

BaGency uses two Google Fonts families:

  • Inter (font-sans) — Body text, UI elements
  • Space Grotesk (font-display) — Headings, display text

To change fonts, update these two locations:

1. Google Fonts link in every HTML file's <head>:

HTML 
<!-- Replace the fonts in the URL -->
<link href="https://fonts.googleapis.com/css2?family=YOUR+FONT:wght@400;500;600;700&family=DISPLAY+FONT:wght@400;600;700&display=swap" rel="stylesheet">

2. Tailwind config in js/tailwind-config.js:

JavaScript — js/tailwind-config.js 
fontFamily: {
    sans: ['Inter', 'sans-serif'],       // ← body font
    display: ['Space Grotesk', 'sans-serif'], // ← heading font
},

Content

BaGency uses two approaches for content:

Dynamic content (data.js): Services, pricing, testimonials, FAQ, team members, and blog posts are stored in js/data.js as JavaScript arrays/objects. Edit these to update content across all pages at once.

JavaScript — js/data.js (example) 
const servicesData = [
    {
        icon: 'message-square',
        title: 'AI Chatbots',              // ← Edit this
        description: '24/7 intelligent...', // ← Edit this
        features: ['NLP', 'Multi-language', 'CRM'],
        color: 'primary',
    },
    // ... more services
];

Static content (HTML): About page text, legal page content, and blog post bodies are written directly in the HTML. Edit the corresponding file to update.

Images

Images are sourced from Unsplash via URL. To use your own images:

  1. Download or create your images and place them in an images/ folder.
  2. Search for the Unsplash URL in data.js (for dynamic content) or the HTML files (for static content).
  3. Replace the URL with a relative path, e.g., images/hero.jpg.

Icons

BaGency uses Lucide Icons (via CDN). Icons are placed using the data-lucide attribute:

HTML — Lucide Icon Usage 
<!-- Icon in HTML -->
<i data-lucide="rocket" class="w-5 h-5"></i>

<!-- Icon in data.js -->
{ icon: 'message-square', ... }

To find icons, visit lucide.dev/icons. The icon name in the URL (e.g., lucide.dev/icons/rocket) is the value you use for data-lucide or the icon property in data.js.

Size is controlled via Tailwind width/height classes: w-4 h-4, w-5 h-5, w-6 h-6, etc.

Dark Mode

BaGency uses Tailwind's class-based dark mode strategy. When the dark class is added to the <html> element, all dark: Tailwind utilities activate.

How It Works

  1. The theme toggle button in the navbar triggers initTheme() in js/main.js.
  2. On click, it toggles the dark class on document.documentElement.
  3. The user's preference is saved to localStorage under the key "theme".
  4. On page load, the saved preference is restored. If none exists, the system's prefers-color-scheme media query is used.

Customizing Dark Mode Colors

Dark mode colors are applied via Tailwind's dark: prefix directly in the HTML classes. For example:

HTML — Dark Mode Classes 
<!-- Background changes on dark mode -->
<body class="bg-white dark:bg-gray-900 text-gray-900 dark:text-white">

<!-- Glass card adapts -->
<div class="glass-card dark:border-gray-700">...</div>

<!-- Text color adapts -->
<p class="text-gray-600 dark:text-gray-400">...</p>

The glassmorphism styles in css/style.css also have .dark scoped variants for backgrounds and borders. If you change the base background color, update both the light and dark variants.

To force dark mode on page load, add class="dark" directly to the <html> tag: <html lang="en" class="dark">.

Animations

BaGency includes a rich animation system powered by CSS and JavaScript.

Scroll Reveal

Elements with the classes .reveal, .reveal-left, .reveal-right, or .reveal-scale automatically animate into view when scrolled into the viewport. This uses IntersectionObserver for performant, jank-free animations.

HTML — Scroll Reveal Classes 
<!-- Fade up -->
<div class="reveal">Fades up from below</div>

<!-- Slide from left -->
<div class="reveal-left">Slides in from left</div>

<!-- Slide from right -->
<div class="reveal-right">Slides in from right</div>

<!-- Scale in -->
<div class="reveal-scale">Scales up into view</div>

<!-- Delayed reveal -->
<div class="reveal" style="transition-delay: 0.2s">Delayed</div>

Adding Custom Animations

To add a new animation, define it in js/tailwind-config.js:

JavaScript — js/tailwind-config.js 
animation: {
    // ... existing animations ...
    'spin-slow': 'spinSlow 3s linear infinite',
},
keyframes: {
    // ... existing keyframes ...
    spinSlow: {
        '0%': { transform: 'rotate(0deg)' },
        '100%': { transform: 'rotate(360deg)' },
    },
},

Then use it in HTML: <div class="animate-spin-slow">...</div>

Deployment

Since BaGency is a static HTML template, it can be deployed to any web server or hosting platform. No server-side processing, databases, or build steps are needed.

Netlify

  1. Create a free account at netlify.com.
  2. Drag and drop the bagency/ folder onto the Netlify dashboard, or connect your Git repository.
  3. If using Git, set the Publish directory to public/bagency (or wherever the folder lives in your repo).
  4. Your site is live! Netlify provides a free .netlify.app subdomain.

Vercel

  1. Sign up at vercel.com.
  2. Import your Git repository.
  3. Set the Root Directory to the folder containing the template files.
  4. Set Framework Preset to "Other".
  5. Click Deploy.

GitHub Pages

  1. Push your code to a GitHub repository.
  2. Go to Settings → Pages.
  3. Select the branch and folder (e.g., main branch, /public/bagency root).
  4. Save — your site will be available at https://username.github.io/repo-name/.

cPanel / Traditional Hosting

  1. Log into your hosting control panel (cPanel, Plesk, etc.).
  2. Open the File Manager or connect via FTP/SFTP.
  3. Navigate to your website's root directory (usually public_html/).
  4. Upload the entire bagency/ folder.
  5. Visit https://yourdomain.com/bagency/.

Any Static Host

BaGency works with any hosting that serves static files: AWS S3 + CloudFront, Firebase Hosting, Cloudflare Pages, DigitalOcean App Platform, etc. Simply upload the files and configure the root directory.

For clean URLs (e.g., yourdomain.com/about instead of yourdomain.com/about.html), most hosts support URL rewriting. On Netlify, create a _redirects file. On Apache, use .htaccess.
